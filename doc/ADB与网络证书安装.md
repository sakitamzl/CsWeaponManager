## 前置要求

在开始之前，请确保满足以下条件：

1. **MuMu模拟器已安装并运行**
2. **MuMu模拟器已开启ADB调试**

> 💡 **说明：** 系统已内置Charles CA证书（有效期至2026年11月），无需手动导出证书文件。如需使用其他抓包工具的证书，请参考[手动安装方法](#手动安装方法)。

## MuMu模拟器配置

### ⚠️ 重要：开启"可选系统盘"功能

这是**最关键**的步骤！如果不开启此功能，证书将无法安装到系统目录。

#### 操作步骤：

1. 打开MuMu模拟器
2. 点击右上角的 **"设置"** 按钮（齿轮图标）
3. 在设置中心找到 **"可选系统盘"** 选项
4. **勾选** "可选系统盘"
5. 点击 **"保存设置"**
6. **重启MuMu模拟器**（必须重启才能生效）

> 💡 **提示：** 开启"可选系统盘"后，系统分区才允许写入操作。

## 自动安装证书

本系统提供了Web界面来自动安装证书，无需手动执行命令。

### 步骤1：访问Dev-Tools页面

1. 启动项目后，访问 `http://localhost:9003`
2. 点击左侧菜单的 **"开发工具"** (Dev-Tools)

### 步骤2：扫描并连接设备

**方式一：自动扫描**
- 点击 **"扫描局域网设备"** 按钮
- 系统会自动扫描局域网内的ADB设备
- 等待扫描完成，选择目标设备

**方式二：手动连接**
- 点击 **"手动连接设备"** 按钮
- 输入设备地址（格式：`IP:端口`）
  - 例如：`10.0.0.1:5555`
- 点击连接

### 步骤3：查看设备信息

连接成功后会显示设备详细信息：

- **设备序列号**
- **连接地址**
- **设备型号**
- **Android版本**
- **SDK版本**
- **Root权限状态** ✓/✗

### 步骤4：检查证书状态

系统会自动检测证书安装状态：

- **✓ 已安装** - 证书已存在于系统目录
- **✗ 未安装** - 需要安装证书

### 步骤5：安装证书

1. 点击 **"安装证书"** 按钮
2. 系统会自动执行以下操作：
   - 计算证书的hash值
   - 推送证书到设备临时目录
   - 使用root权限复制到系统证书目录
   - 设置正确的权限（644）
   - 验证安装结果
3. 等待安装完成提示

### 步骤6：重启模拟器

证书安装成功后，**必须重启MuMu模拟器**使证书生效。

## 常见问题排查

### 问题1：提示"系统分区为只读，无法复制证书"

**错误信息：**
```
✗ 系统分区为只读，无法复制证书
cp: /system/etc/security/cacerts/xxx.0: Read-only file system
```

**原因：** 
未开启MuMu模拟器的"可选系统盘"功能

**解决方案：**
1. 按照[MuMu模拟器配置](#mumu模拟器配置)章节的步骤开启"可选系统盘"
2. 保存设置并重启模拟器
3. 重新连接设备并尝试安装证书

---

### 问题2：提示"设备没有root权限"

**错误信息：**
```
✗ 设备没有root权限，无法安装系统证书
```

**原因：** 
MuMu模拟器的root权限未授予给ADB

**解决方案：**
1. 在模拟器屏幕上会弹出**超级用户授权提示**
2. 点击 **"允许"** 授予root权限
3. 如果没有弹出提示，检查模拟器的root管理应用
4. 重新尝试安装证书

---

### 问题3：证书安装成功但抓包失败

**症状：**
- 证书状态显示"已安装"
- 但抓包工具无法拦截HTTPS流量

**可能原因：**
1. 证书未生效（未重启模拟器）
2. 网络代理未配置
3. 抓包工具未开启SSL拦截

**解决方案：**

**步骤1：确保已重启模拟器**
- 完全关闭并重新启动MuMu模拟器

**步骤2：配置模拟器网络代理**
1. 在模拟器中打开 **设置 → 网络和互联网 → WLAN**
2. 长按已连接的WiFi，选择 **修改网络**
3. 展开 **高级选项**
4. 代理设置选择 **手动**
5. 填入代理信息：
   - **主机名：** 电脑的IP地址（如：192.168.123.250）
   - **端口：** 抓包工具的端口（Charles默认8888，Fiddler默认8888）
6. 在 **对以下网址不使用代理** 中填入：`10.0.2.2`
7. 保存设置

**步骤3：配置抓包工具**

**Charles：**
1. 打开 **Proxy → SSL Proxying Settings**
2. 勾选 **Enable SSL Proxying**
3. 点击 **Add**，添加：
   - Host: `*`
   - Port: `*`
4. 点击 **OK** 保存

**Fiddler：**
1. 打开 **Tools → Options → HTTPS**
2. 勾选 **Decrypt HTTPS traffic**
3. 勾选 **Ignore server certificate errors**

---

### 问题4：无法连接到设备

**错误信息：**
```
✗ 无法连接到设备: 192.168.123.137:5555
```

**可能原因：**
1. MuMu模拟器未启动
2. ADB端口未开放
3. 防火墙阻止连接
4. 设备不在同一局域网

**解决方案：**

1. **确保MuMu模拟器已启动**
   - 模拟器必须处于运行状态

2. **检查ADB端口**
   - MuMu模拟器默认ADB端口：`5555`
   - 确认端口未被占用

3. **检查防火墙设置**
   - Windows防火墙可能阻止5555端口
   - 临时关闭防火墙测试，或添加端口例外

4. **确认网络连接**
   - 确保电脑和模拟器在同一局域网
   - 检查IP地址是否正确

5. **手动测试ADB连接**
   ```bash
   # 如果安装了adb命令行工具
   adb connect 192.168.123.137:5555
   adb devices
   ```

---

### 问题5：证书hash计算失败

**错误信息：**
```
✗ 计算证书hash失败
```

**原因：**
证书文件格式不正确或损坏

**解决方案：**
1. 确保证书是PEM格式
2. 重新从抓包工具导出证书
3. 检查证书文件内容是否完整

## 手动安装方法

如果自动安装失败，可以使用命令行手动安装。

### 前提条件

- 已安装 `adb` 命令行工具
- 已安装 `openssl` 工具

### 步骤1：导出证书

**Charles：**
1. 打开Charles
2. 菜单栏：**Help → SSL Proxying → Save Charles Root Certificate**
3. 保存为 `charles-proxy.pem`

**Fiddler：**
1. 打开Fiddler
2. 菜单栏：**Tools → Options → HTTPS → Actions → Export Root Certificate to Desktop**
3. 保存为 `fiddler-root.cer`
4. 转换为PEM格式：
   ```bash
   openssl x509 -inform DER -in fiddler-root.cer -out fiddler-root.pem
   ```

### 步骤2：计算证书hash

```bash
openssl x509 -subject_hash_old -in charles-proxy.pem
```

输出示例：
```
a6391e34
```

### 步骤3：重命名证书

将证书重命名为 `{hash}.0`：

```bash
# Linux/Mac
cp charles-proxy.pem a6391e34.0

# Windows
copy charles-proxy.pem a6391e34.0
```

### 步骤4：连接设备

```bash
adb connect 192.168.123.137:5555
```

### 步骤5：获取root权限

```bash
adb root
```

> 💡 **注意：** 模拟器中会弹出超级用户授权提示，点击"允许"

### 步骤6：推送证书

```bash
adb push a6391e34.0 /system/etc/security/cacerts/
```

### 步骤7：设置权限

```bash
adb shell "chmod 644 /system/etc/security/cacerts/a6391e34.0"
```

### 步骤8：验证安装

```bash
adb shell "ls -la /system/etc/security/cacerts/a6391e34.0"
```

输出示例：
```
-rw-r--r-- 1 root root 1234 2025-11-08 15:00 a6391e34.0
```

### 步骤9：重启模拟器

证书安装完成后，重启MuMu模拟器使证书生效。

## 常见问题 FAQ

**Q: 为什么必须开启"可选系统盘"？**

A: MuMu模拟器默认系统分区是只读的，开启"可选系统盘"后才允许写入系统文件。

**Q: 证书安装后需要重启吗？**

A: 是的，必须重启模拟器才能使证书生效。

**Q: 支持其他Android设备吗？**

A: 支持，但需要设备已root且系统分区可写。

**Q: 证书会过期吗？**

A: 会的，证书有有效期。系统内置的Charles证书有效期至2026年11月，过期后需要重新安装新证书。
