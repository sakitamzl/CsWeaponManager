# 库存组件使用指南

> 本文档介绍如何使用库存组件功能，包括查看、管理和更新Steam库存中的CS2组件物品。

## ⚠️ 先决条件

在使用库存组件功能之前，请确保：

✅ **已配置完美世界APP数据源** - 用于获取组件内的物品详情

> 📖 **重要提示：** 库存组件功能需要通过完美世界APP来获取组件内的具体物品信息。如果未配置完美世界APP数据源，将无法更新和查看组件内的物品详情。
>
> 💡 **数据源配置教程：** 请前往 **[其他功能 → 数据来源](http://localhost:9003/settings/data-source)** 页面添加对应账号的完美世界APP数据源。详细配置步骤请参考文档 [`Documents\其他功能\数据来源\数据来源使用指南.md`](其他功能/数据来源/数据来源使用指南.md)

如果您尚未完成数据源配置，请先按照上述文档完成配置后再继续。

## 功能介绍

库存组件功能用于管理Steam库存中的CS2组件物品（Storage Container）。组件是CS2中的特殊物品，可以存储多个其他饰品。

**核心功能：**
- 📦 **组件管理** - 查看和管理Steam账号下的所有库存组件
- 💰 **价格跟踪** - 记录购入价格，对比各平台参考价格
- 📊 **统计分析** - 查看总价值、盈亏情况等统计数据
- 🔄 **数据同步** - 从完美世界APP获取组件内的物品数据
- 🎨 **多种显示** - 支持组合模式和明细模式查看
- 🏷️ **物品详情** - 查看磨损值、贴纸、挂件、改名等详细信息

## 界面功能详解

### 账号选择

在页面顶部的第一个下拉框中选择Steam账号：

- 下拉框显示格式：`数据名称 (SteamID) - X件`
- 数字表示该账号下的组件数量
- 支持搜索过滤账号

**操作步骤：**
1. 点击 **"选择Steam账号"** 下拉框
2. 从列表中选择目标账号
3. 系统会自动加载该账号的组件数据

### 组件选择

在第二个下拉框中可以选择特定组件查看：

- 显示格式：`物品名称 (数量:X)`
- 同时显示 assetid 用于精确定位
- 支持搜索和清空选择

**使用场景：**
- 查看单个组件的详细信息
- 更新特定组件的物品数据
- 快速定位某个组件

### 搜索功能

使用搜索框可以按物品名称过滤组件：

1. 在 **"搜索武器名称..."** 输入框中输入关键词
2. 点击 **"搜索"** 按钮或按 `Enter` 键
3. 点击 **"重置"** 按钮清空搜索条件

**搜索技巧：**
- 支持模糊搜索
- 可以搜索武器名、皮肤名
- 搜索不区分大小写

### 显示模式

页面提供两种显示模式：

#### 🔹 组合模式（默认）

- 相同物品合并显示
- 显示该物品的总数量
- 显示所有价格的总和
- 点击行可展开查看详细列表

**适用场景：**
- 快速了解组件中有哪些物品
- 查看各类物品的总价值
- 批量查看相同物品

#### 🔹 明细模式

- 每个物品单独显示一行
- 显示完整的物品信息
- 包含所属组件和组件ID

**适用场景：**
- 查看每个物品的具体信息
- 区分相同物品的不同磨损值
- 精确定位某个物品

**切换方式：**
使用右上角的开关按钮切换：`组合模式 ⇄ 明细模式`

## 查看统计数据

页面顶部显示5个统计卡片，实时展示组件价值：

### 📊 统计指标

| 指标 | 说明 |
|------|------|
| **总组件数量** | 当前账号下所有组件中的物品总数 |
| **购入总价值** | 所有物品的购入价格总和（手动录入） |
| **悠悠有品最低价** | 所有物品在悠悠有品的最低价总和 |
| **BUFF最低价** | 所有物品在网易BUFF的最低价总和 |
| **Steam参考价** | 所有物品在Steam市场的参考价总和 |

### 💹 盈亏显示

每个平台价格下方会显示与购入价的差价：
- **红色 +金额** - 亏损（平台价格 > 购入价格）
- **绿色 -金额** - 盈利（平台价格 < 购入价格）

**示例：**
```
悠悠有品最低价
¥1,234.56
+¥123.45  ← 相比购入价亏损123.45元
```

## 更新组件数据

### 更新单个组件

**适用场景：** 更新特定组件内的物品信息

**操作步骤：**
1. 选择Steam账号
2. 在第二个下拉框中选择要更新的组件
3. 点击 **"获取/更新组件物品"** 按钮
4. 等待更新完成（显示更新物品数量）

**说明：**
- 按钮在未选择组件时为禁用状态
- 更新成功后会自动同步购入价格（从历史交易记录）
- 更新可能需要10-30秒

### 批量更新组件

**适用场景：** 一次性更新账号下所有组件

**操作步骤：**
1. 选择Steam账号
2. 点击 **"获取/更新全部组件"** 按钮
3. 在确认对话框中点击 **"确定"**
4. 等待批量更新完成

**注意事项：**
- ⚠️ 批量更新可能需要较长时间（数分钟到十几分钟）
- 系统会自动过滤掉数量为0的组件
- 更新成功后显示成功/失败统计
- 会自动同步购入价格

### 更新平台价格

**适用场景：** 获取最新的平台参考价格

**操作步骤：**
1. 选择Steam账号
2. 点击 **"获取/更新平台价格"** 按钮
3. 等待价格更新完成

**说明：**
- 会同时更新悠悠有品和BUFF的价格
- 强制重新获取最新价格（不使用缓存）
- 更新完成后统计数据会自动刷新

## 价格管理

### 手动编辑购入价格

**适用场景：** 手动录入或修正物品的购入价格

**操作步骤：**
1. 在表格中找到目标物品
2. 点击 **"购入价格"** 列的单元格
3. 输入新的价格金额
4. 按 `Enter` 键保存，或按 `Esc` 键取消

**编辑特性：**
- ✨ 点击价格单元格即可编辑
- 💾 自动保存到数据库
- 📊 实时更新统计数据
- 🔄 支持取消编辑（Esc键）

**注意事项：**
- 价格单位为人民币（¥）
- 建议保留两位小数
- 组合模式下编辑会应用到所有同类物品

## 查看物品详情

### 组合模式详情

在组合模式下，点击数量大于1的物品行可以展开详细列表：

#### 📋 展开列表内容

每个展开的物品卡片显示：
- **物品图片** - 带贴纸和挂件覆盖层
- **磨损值** - 数值和可视化进度条
- **价格信息** - 购入价、悠悠价、BUFF价、Steam价
- **改名标签** - 如果物品被改名

#### 🎨 磨损值显示

磨损值使用颜色渐变条表示：
- 🟢 **崭新出厂 (FN)** - 0.00 ~ 0.07 - 绿色
- 🟡 **略有磨损 (MW)** - 0.07 ~ 0.15 - 浅绿
- 🟠 **久经沙场 (FT)** - 0.15 ~ 0.38 - 橙黄
- 🔴 **破损不堪 (WW)** - 0.38 ~ 0.45 - 橙红
- 🔴 **战痕累累 (BS)** - 0.45 ~ 1.00 - 红色

白色指针指示当前磨损值位置。

#### 📄 分页功能

当展开的物品超过12个时，会显示分页器：
- 每页显示12个物品卡片
- 分页器位置：物品名称下方
- 支持快速跳转到指定页

### 明细模式详情

明细模式下每行显示完整的物品信息：

| 列名 | 说明 |
|------|------|
| **图片** | 物品的图片预览 |
| **物品名称** | 武器名 \| 皮肤名（磨损范围）<br>下方显示贴纸、挂件、改名 |
| **武器类型** | 如：步枪、手枪、刀 |
| **武器等级** | 如：隐秘、受限、军规级 |
| **所属组件** | 组件的 assetid |
| **组件ID** | 物品的 goods_assetid |
| **数量** | 该物品的数量 |
| **磨损值** | 数值和可视化进度条 |
| **磨损范围** | 如：崭新出厂、久经沙场 |
| **购入价格** | 可点击编辑 |
| **悠悠价格** | 悠悠有品参考价 |
| **BUFF价格** | 网易BUFF参考价 |
| **Steam价格** | Steam市场参考价 |

#### 🎯 物品名称详情

物品名称下方可能包含：
- **印花（贴纸）** - 以小图标显示，鼠标悬停查看名称
- **挂件** - 显示挂件图标
- **改名** - 显示自定义名称

#### 🖱️ 交互操作

- 点击磨损值列可排序
- 点击价格列可排序
- 支持点击行查看更多信息

|------|------|------|
| `steam_hash_name` | String | Steam市场物品名称 |
| `item_name` | String | 物品显示名称 |

| `assetid` | String | 所属组件ID |
| `goods_assetid` | String | 物品的唯一ID |

| `float_range` | String | 磨损范围文字描述 |
| `buy_price` | Float | 购入价格（元） |
| `yyyp_price` | Float | 悠悠有品价格（元） |
| `buff_price` | Float | 网易BUFF价格（元） |

| 字段 | 说明 |

| `total_buff_price` | 所有该物品的BUFF价总和 |
| `total_steam_price` | 所有该物品的Steam价总和 |
| `goods_assetids` | 所有物品的ID数组 |
| `weapon_floats` | 所有物品的磨损值数组 |
| `buy_prices` | 所有物品的购入价数组 |
| `yyyp_prices` | 所有物品的悠悠价数组 |
| `buff_prices` | 所有物品的BUFF价数组 |
| `steam_prices` | 所有物品的Steam价数组 |
| `stickers` | 所有物品的贴纸数组 |
| `pendants` | 所有物品的挂件数组 |
| `renames` | 所有物品的改名数组 |

## 常见问题

### 🔸 为什么看不到组件数据？

**可能原因：**
1. 还没有添加Steam账号
2. 该账号的库存中没有组件
3. 还没有更新组件数据

**解决方法：**
1. 前往 **"数据来源"** 页面添加Steam数据源
2. 前往 **"Steam库存"** 页面同步库存
3. 点击 **"获取/更新全部组件"** 按钮

### 🔸 组件数量显示为0怎么办？

**原因：** 组件数量是指组件内的物品数量，不是组件本身的数量

**解决方法：**
1. 选择该组件
2. 点击 **"获取/更新组件物品"** 按钮
3. 等待系统从完美世界APP获取数据

### 🔸 平台价格显示0.00？

**原因：** 还没有获取平台价格数据

**解决方法：**
1. 点击 **"获取/更新平台价格"** 按钮
2. 等待系统从悠悠有品和BUFF获取最新价格
3. 如果某些物品在平台上不存在，价格仍会显示0.00

### 🔸 更新组件失败？

**可能原因：**
1. 完美世界APP数据源未配置
2. Cookie已过期
3. 网络连接问题

**解决方法：**
1. 检查 **"数据来源"** 中的完美世界APP配置
2. 更新Cookie（参考数据来源文档）
3. 检查网络连接和代理设置

### 🔸 购入价格为什么是0？

**原因：** 系统无法从历史交易记录中找到对应物品

**解决方法：**
1. 点击价格单元格手动输入购入价格
2. 确保已同步 **"交易记录"** 数据
3. 如果是新购入物品，需要手动录入

### 🔸 组合模式下看不到具体磨损值？

**原因：** 组合模式将相同物品合并显示，磨损值各不相同

**解决方法：**
- **方式一：** 点击该行展开详细列表，查看每个物品的磨损值
- **方式二：** 切换到明细模式，每个物品单独显示

### 🔸 贴纸或挂件图片无法显示？

**可能原因：**
1. 图片文件不存在
2. 图片路径不正确
3. hashName格式不匹配

**说明：**
- 系统会自动根据hashName生成图片路径
- 图片路径：`/weapon_imgs/Sticker___[hashName].png`
- 如果图片不存在，显示占位符

### 🔸 展开的物品列表为什么有分页？

**原因：** 相同物品超过12个时自动分页，避免页面过长

**操作：**
- 使用物品名称下方的分页器切换页码
- 每页显示12个物品卡片
- 支持点击页码快速跳转

### 🔸 如何快速找到某个组件？

**方法一：使用组件选择下拉框**
1. 点击第二个下拉框
2. 在搜索框中输入组件名称或assetid
3. 选择目标组件

**方法二：使用搜索功能**
1. 在搜索框中输入物品关键词
2. 点击搜索按钮
3. 从结果中找到目标组件

### 🔸 统计数据不准确？

**可能原因：**
1. 部分物品没有购入价格
2. 平台价格未更新
3. 数据未及时刷新

**解决方法：**
1. 补全所有物品的购入价格
2. 点击 **"获取/更新平台价格"** 更新最新价格
3. 刷新页面重新加载数据

## 💡 使用技巧

### 快速批量操作

1. **批量更新全部数据：**
   ```
   获取/更新全部组件 → 等待完成 → 获取/更新平台价格
   ```

2. **快速录入价格：**
   - 使用Tab键在编辑框间快速切换
   - 录入价格后按Enter自动保存并跳到下一行

3. **高效筛选数据：**
   - 组合模式查看概览
   - 搜索定位目标物品
   - 切换明细模式查看详情

### 数据分析技巧

1. **盈亏分析：**
   - 查看统计卡片的差价显示
   - 对比不同平台的价格差异
   - 明细模式按价格排序查看

2. **物品管理：**
   - 组合模式快速了解库存构成
   - 展开查看同类物品的磨损值分布
   - 记录购入价格便于后续分析

3. **定期维护：**
   - 定期更新平台价格（建议每周）
   - 及时录入新购入物品的价格
   - 批量更新组件数据保持同步
