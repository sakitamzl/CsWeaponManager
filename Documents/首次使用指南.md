## 系统要求

### 必需环境
- **操作系统**: Windows 7/10/11
- **Python**: 3.8 或更高版本（开发版本需要）
- **浏览器**: Chrome、Edge 或 Firefox（用于获取Cookie）

## 安装步骤

### 方式一：使用发行版（推荐新手）

1. **下载发行版**
   - 从 [Releases](../Releases) 目录下载最新版本的 `CsWeaponManager.zip`
   - 解压到任意目录（建议路径不包含中文）

2. **下载武器图片资源包**
   - 访问 [Google Drive](https://drive.google.com/file/d/1Jz50N_op51l2xjMPzFX5SKO9rQMNr4nP/view?usp=drive_link)
   - 下载 `weapon_imgs.zip` 并解压到项目根目录
   - 确保目录结构为：`CsWeaponManager/weapon_imgs/`

3. **检查文件结构**
   ```
   CsWeaponManager/
   ├── weapon_imgs/         # 武器图片资源（必需）
   ├── WebSite/             # 前端网站
   ├── backEnd.exe          # 后端服务
   ├── Spider.exe           # 爬虫服务
   ├── WebServer.exe        # 网页服务
   ├── conf.ini             # 配置文件
   ├── start_all.bat        # 启动脚本
   └── csweaponmanager.db   # 数据库（首次启动自动生成）
   ```

### 方式二：从源码运行（开发者）

1. **克隆项目**
   ```bash
   git clone [项目地址]
   cd CsWeaponManager
   ```

2. **安装依赖**
   ```bash
   pip install -r requirements.txt
   ```

3. **下载图片资源**（同上）

## 配置说明

### 1. 代理配置（重要）

访问Steam需要配置代理，编辑 `conf.ini` 文件：

```ini
[http_proxy]
true = True              # 是否启用HTTP代理
host = 127.0.0.1        # 代理服务器地址
port = 10811            # HTTP代理端口

[socks5]
host = 127.0.0.1        # SOCKS5代理地址
port = 10808            # SOCKS5代理端口
```

**配置步骤：**
1. 启动你的代理软件（如v2rayNG、Clash等）
2. 查看代理软件的本地监听端口
3. 将端口号填入 `conf.ini` 对应位置
4. 常见端口：HTTP代理通常为 `10809`、`10811`，SOCKS5通常为 `10808`

### 2. 日志配置

```ini
[LogLevel]
level = error           # 日志级别：debug/info/warning/error
sometime = 14           # 日志保留天数
```

**建议设置：**
- 日常使用：`level = error`
- 调试问题：`level = debug`

### 3. 数据库配置

```ini
[database]
sqlite_file = csweaponmanager.db
```

默认配置即可，数据库文件会自动创建在项目根目录。

## 启动系统

### 发行版启动

1. **双击运行 `start_all.bat`**
   - 系统会自动启动三个服务：后端、爬虫、网页服务
   - 等待约10-30秒，直到看到"服务启动成功"提示

2. **访问Web界面**
   - 打开浏览器，访问：`http://localhost:5173`
   - 如果端口被占用，系统会自动使用其他端口，请查看启动窗口的提示

### 源码版启动

1. **激活Conda环境**（如果使用）
   ```bash
   conda activate CS2DB
   ```

2. **运行启动脚本**
   ```bash
   start.bat
   ```

### 验证启动成功

- 浏览器能正常打开系统主页
- 主页显示系统各模块状态
- 没有明显的错误提示

## Cookie配置

Cookie是系统访问各交易平台的凭证，**必须配置**才能正常使用。

### 1. 进入数据源配置页面

在系统主页点击 **"数据源"** 或 **"设置"** 菜单。

### 2. 配置Steam Cookie

**获取步骤：**
1. 打开Chrome浏览器，登录 [Steam社区](https://steamcommunity.com)
2. 按 `F12` 打开开发者工具
3. 切换到 **"Application"** 或 **"应用"** 标签
4. 左侧选择 **"Cookies"** → **"https://steamcommunity.com"**
5. 找到以下Cookie并复制其值：
   - `steamLoginSecure`（必需）
   - `sessionid`（必需）
6. 将Cookie值粘贴到系统的Steam配置框中
7. 点击 **"保存"** 并 **"测试连接"**

### 3. 配置悠悠有品Cookie

**获取步骤：**
1. 登录 [悠悠有品](https://www.youpin898.com)
2. 按 `F12` 打开开发者工具
3. 切换到 **"Network"** 或 **"网络"** 标签
4. 刷新页面，点击任意请求
5. 在 **"Headers"** 中找到 **"Cookie"** 字段
6. 复制完整的Cookie值
7. 粘贴到系统的悠悠有品配置框中

### 4. 配置BUFF Cookie

**获取步骤：**
1. 登录 [BUFF163](https://buff.163.com)
2. 按 `F12` 打开开发者工具
3. 找到 `session` Cookie
4. 复制并粘贴到系统配置中

### 5. 配置CSFloat Cookie

**获取步骤：**
1. 登录 [CSFloat](https://csfloat.com)
2. 按 `F12` 打开开发者工具
3. 找到认证相关的Cookie
4. 复制并粘贴到系统配置中

**注意事项：**
- Cookie有时效性，过期后需要重新获取
- 系统支持Steam Cookie自动刷新（需在自动化管理中开启）
- 建议定期检查Cookie状态

## 基本使用

### 1. 同步Steam库存

1. 进入 **"库存管理"** 页面
2. 点击 **"同步Steam库存"** 按钮
3. 等待同步完成（首次同步可能需要几分钟）
4. 查看库存列表，确认饰品已正确显示

### 2. 查看价格对比

1. 在库存列表中选择任意饰品
2. 点击 **"价格对比"** 按钮
3. 系统会显示该饰品在各平台的价格：
   - Steam市场价格
   - 悠悠有品价格
   - BUFF价格
   - CSFloat价格

### 3. 记录购买

1. 进入 **"购买记录"** 页面
2. 点击 **"添加购买记录"** 按钮
3. 填写购买信息：
   - 饰品名称
   - 购买价格
   - 购买平台
   - 购买时间
4. 系统会自动从数据库匹配价格信息

### 4. 记录出售

1. 进入 **"出售管理"** 页面
2. 点击 **"添加出售记录"**
3. 填写出售信息
4. 系统会自动计算收益

### 5. 查看数据统计

1. 进入 **"数据统计"** 页面
2. 查看以下信息：
   - 总收益统计
   - 各平台交易分布
   - 饰品类型分析
   - 收益趋势图表

### 6. 租赁管理

1. 进入 **"租赁列表"** 页面
2. 查看当前出租的饰品
3. 系统会自动同步悠悠有品和BUFF的租赁状态
4. 租赁被买断时会自动同步到出售记录

### 7. 库存组件查询

1. 进入 **"库存组件"** 页面
2. 查看Steam库存历史记录
3. 支持以下视图：
   - 列表视图
   - 组合视图
   - 组合列表视图
4. 可查看饰品的印花、挂件、改名等详细信息

## 常见问题

### Q1: 启动后无法访问Web界面？

**解决方案：**
- 检查端口是否被占用（默认5173）
- 查看启动窗口是否有错误提示
- 尝试访问 `http://127.0.0.1:5173`
- 检查防火墙设置

### Q2: Steam数据无法同步？

**可能原因：**
1. **代理未配置或配置错误**
   - 检查 `conf.ini` 中的代理设置
   - 确认代理软件正在运行
   - 测试代理是否能访问Steam

2. **Cookie过期**
   - 重新获取Steam Cookie
   - 在数据源页面更新Cookie

3. **网络问题**
   - 检查网络连接
   - 尝试更换代理节点

### Q3: 图片无法显示？

**解决方案：**
- 确认已下载 `weapon_imgs` 图片资源包
- 检查图片目录是否在项目根目录
- 系统会自动从悠悠有品下载缺失的图片

### Q4: 数据库错误？

**解决方案：**
- 备份 `csweaponmanager.db` 文件
- 删除数据库文件，重新启动系统（会重新创建）
- 如需保留数据，使用SQLite工具修复数据库

### Q5: Cookie如何自动刷新？

**设置步骤：**
1. 进入 **"自动化管理"** 页面
2. 找到 **"Steam Cookie自动刷新"** 选项
3. 开启自动刷新功能
4. 系统会定期自动更新Cookie

### Q6: 如何备份数据？

**备份内容：**
- `csweaponmanager.db` - 数据库文件（最重要）
- `conf.ini` - 配置文件
- `log/` - 日志文件（可选）

**备份方法：**
- 定期复制数据库文件到安全位置
- 建议每周备份一次

### Q7: 版本更新后数据丢失？

**注意事项：**
- 更新前务必备份数据库
- 某些大版本更新（如v1.x到v2.x）可能不兼容
- 查看 [更新日志](updateLog.md) 了解数据迁移说明
- 按照更新日志中的SQL语句执行数据迁移

### Q8: 系统运行缓慢？

**优化建议：**
- 定期清理日志文件（超过设定天数会自动清理）
- 减少同时运行的自动化任务
- 调整日志级别为 `error`
- 关闭不需要的平台数据同步
