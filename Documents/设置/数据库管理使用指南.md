## 功能介绍

数据库管理器是一个类似Navicat的可视化数据库管理工具，提供以下功能：

- 📊 **数据查看与编辑** - 可视化查看和编辑表数据
- 🔍 **表结构查看** - 查看表字段、类型、约束等信息
- 💻 **SQL查询** - 执行自定义SQL查询
- 📝 **数据增删改** - 新增、编辑、删除数据行
- 📤 **数据导出** - 导出表数据为CSV格式
- 🔖 **查询保存** - 保存常用SQL查询

---

## 访问数据库管理器

1. 启动项目后，访问 `http://localhost:9003`
2. 点击左侧菜单：**设置 → 数据库管理**
3. 或直接访问：`http://localhost:9003/settings/database-manager`

## 界面布局

数据库管理器采用左右分栏布局：

### 左侧：表列表面板

- **搜索框** - 快速搜索表名
- **表列表** - 显示所有数据表及行数
- **刷新按钮** - 重新加载表列表
- **折叠按钮** - 折叠/展开侧边栏

### 右侧：主内容区

包含三个标签页：

1. **数据** - 查看和编辑表数据
2. **结构** - 查看表结构信息
3. **查询** - 执行SQL查询

## 数据管理

### 查看表数据

**步骤1：选择数据表**

在左侧表列表中点击要查看的表名。

**步骤2：浏览数据**

- 数据以表格形式展示
- 支持横向和纵向滚动
- 每列显示字段名和值
- 底部显示分页控件

### 搜索数据

在数据工具栏右侧的搜索框中输入关键词，系统会在所有字段中搜索匹配的数据。

### 新增数据

**步骤1：点击"新增"按钮**

在数据工具栏点击 **"新增"** 按钮。

**步骤2：填写数据**

在弹出的对话框中填写各字段的值。

**步骤3：保存**

点击 **"保存"** 按钮完成新增。

> 💡 **提示：** 主键字段如果是自增的，可以留空由系统自动生成。

### 编辑数据

**步骤1：点击"编辑"按钮**

在要编辑的数据行右侧点击 **"编辑"** 按钮。

**步骤2：修改数据**

在弹出的对话框中修改字段值。

**步骤3：保存**

点击 **"保存"** 按钮完成修改。

### 删除数据

**单行删除：**
1. 在要删除的数据行右侧点击 **"删除"** 按钮
2. 确认删除操作

**批量删除：**
1. 勾选要删除的数据行（表格左侧复选框）
2. 点击工具栏的 **"删除选中"** 按钮
3. 确认删除操作

> ⚠️ **警告：** 删除操作不可恢复，请谨慎操作！

### 导出数据

点击工具栏的 **"导出"** 按钮，当前表的所有数据将导出为CSV文件并自动下载。

### 刷新数据

点击工具栏的 **"刷新"** 按钮，重新加载当前表的最新数据。

### 分页设置

在数据工具栏右侧的分页控件中：
- 选择每页显示条数（10/20/50/100）
- 点击页码切换页面
- 使用上一页/下一页按钮导航

## 表结构查看

### 查看字段信息

切换到 **"结构"** 标签页，可以看到表的完整结构信息：

**字段列表包含：**
- **ID** - 字段序号
- **字段名** - 字段的名称
- **类型** - 数据类型（如：INTEGER, TEXT, REAL等）
- **非空** - 是否允许为空
- **默认值** - 字段的默认值
- **主键** - 是否为主键字段

### 查看表信息

在字段列表下方的"表信息"区域显示：
- 表名
- 字段数量
- 数据行数
- 创建时间

### 查看建表语句

在"建表语句"区域显示完整的CREATE TABLE SQL语句。

**复制SQL：**
点击 **"复制SQL"** 按钮，将建表语句复制到剪贴板。

## SQL查询

### 执行查询

**步骤1：切换到查询标签**

点击 **"查询"** 标签切换到SQL查询视图。

**步骤2：输入SQL语句**

在查询编辑器中输入SQL语句，例如：

```sql
SELECT * FROM weapons WHERE price > 100;
```

**步骤3：执行查询**

点击 **"执行"** 按钮或按 `F5` 键执行查询。

**步骤4：查看结果**

查询结果会显示在下方的结果区域，包括：
- 返回的行数
- 执行时间
- 结果数据表格

### 支持的SQL语句

**查询语句（SELECT）：**
```sql
-- 查询所有数据
SELECT * FROM table_name;

-- 条件查询
SELECT * FROM table_name WHERE column = 'value';

-- 排序查询
SELECT * FROM table_name ORDER BY column DESC;

-- 聚合查询
SELECT COUNT(*), AVG(price) FROM table_name;
```

**更新语句（UPDATE）：**
```sql
UPDATE table_name 
SET column1 = 'value1', column2 = 'value2' 
WHERE condition;
```

**删除语句（DELETE）：**
```sql
DELETE FROM table_name WHERE condition;
```

**插入语句（INSERT）：**
```sql
INSERT INTO table_name (column1, column2) 
VALUES ('value1', 'value2');
```

> ⚠️ **警告：** UPDATE和DELETE语句会直接修改数据库，请务必添加WHERE条件，避免误操作！

### 查询工具

**清空查询：**
点击 **"清空"** 按钮清空查询编辑器中的内容。

**格式化查询：**
点击 **"格式化"** 按钮自动格式化SQL语句，使其更易读。

**保存查询：**
1. 点击 **"保存查询"** 按钮
2. 输入查询名称
3. 点击 **"保存"**

**加载已保存的查询：**
在工具栏右侧的下拉框中选择已保存的查询，自动加载到编辑器。

### 查询结果

**成功执行：**
- 显示返回的行数
- 显示执行时间（毫秒）
- 以表格形式展示结果

**执行失败：**
- 显示错误信息
- 提示错误原因

## 常见操作

### 快速查找表

在左侧搜索框中输入表名关键词，快速定位到目标表。

### 查看表的所有数据

1. 在左侧点击表名
2. 系统自动切换到"数据"标签
3. 浏览表格数据

使用SQL查询功能执行UPDATE语句：

```sql

WHERE condition;
```

### 统计数据

使用SQL查询功能执行聚合查询：

```sql
SELECT 
  COUNT(*) as total,
  AVG(price) as avg_price,
  MAX(price) as max_price
FROM weapons;
```

1. 切换到"结构"标签
2. 点击"复制SQL"按钮
3. 在查询标签中修改表名后执行

## 常见问题

### Q1: 为什么看不到某些表？

**A:** 可能的原因：
- 表名被搜索过滤了，清空搜索框
- 表列表未刷新，点击刷新按钮
- 数据库连接异常，检查后端服务

### Q2: 编辑数据后为什么没有生效？

**A:** 请检查：
- 是否点击了"保存"按钮
- 是否有错误提示信息
- 字段值是否符合数据类型要求
- 是否违反了唯一性约束

### Q3: SQL查询报错怎么办？

**A:** 常见错误：
- **语法错误** - 检查SQL语句是否正确
- **表名错误** - 确认表名是否存在
- **字段名错误** - 确认字段名是否正确
- **权限不足** - 某些操作可能需要特殊权限

### Q4: 如何备份数据？

**A:** 
1. 使用"导出"功能导出单个表
2. 或使用系统的数据库备份功能（如果有）
3. 建议定期备份重要数据

### Q5: 删除的数据可以恢复吗？

**A:** 不可以。删除操作是永久性的，无法恢复。建议：
- 删除前仔细确认
- 重要数据先备份
- 使用WHERE条件限制删除范围

### Q6: 如何提高查询性能？

**A:** 建议：
- 使用WHERE条件限制查询范围
- 避免SELECT *，只查询需要的字段
- 为常用查询字段建立索引
- 使用LIMIT限制返回行数

### Q7: 分页显示不全怎么办？

**A:** 
- 调整每页显示条数（10/20/50/100）
- 使用搜索功能过滤数据
- 使用SQL查询精确查找

### Q8: 可以同时编辑多行数据吗？

**A:** 不可以直接批量编辑。建议：
- 使用SQL UPDATE语句批量修改
- 或逐行编辑
- 或导出数据，批量修改后重新导入
