# 数据来源使用指南

> 本文档介绍如何配置和使用CS饰品价格数据来源，包括Steam市场、网易BUFF、悠悠有品等平台。

本文档的操作流程基于 **MuMu模拟器** 实现。在开始使用数据来源功能之前，请确保：

1. ✅ **已安装并配置MuMu模拟器**（版本 ≥ 5.5.0）
2. ✅ **已完成CA证书安装**（用于抓取APP数据）

> 📖 **CA证书安装教程：** 请参考文档 [`Documents\其他功能\开荒工具\ADB与网络证书安装.md`](../其他功能/开荒工具/ADB与网络证书安装.md)

## 功能介绍

数据来源功能允许您配置多个CS饰品价格数据源，系统会自动从这些平台采集最新的饰品价格信息。

**支持的平台：**
- 🎮 **Steam市场** - 官方市场价格（支持登录/未登录）
- 💰 **网易BUFF** - 国内主流交易平台
- 🎯 **悠悠有品** - C5GAME旗下交易平台
- 🌟 **完美世界APP** - 官方国服平台

---

## 访问数据来源页面

1. 启动项目后，访问 `http://localhost:9003`
2. 点击左侧菜单：**设置 → 数据来源**
3. 或直接访问：`http://localhost:9003/settings/data-source`

---

## 创建数据源

### 第一次使用

如果还没有任何数据源，页面会显示空状态提示：

1. 点击 **"创建第一个数据源"** 按钮
2. 在弹出的对话框中输入 **SteamID**（用于分组管理）
3. 选择要创建的 **数据源类型**
4. 按照对应类型的配置步骤完成设置

### 添加更多数据源

已有数据源后，可以通过以下方式添加：

**方式一：在现有SteamID分组下添加**
- 点击分组右上角的 **"添加数据源"** 按钮
- 选择数据源类型并配置

**方式二：创建新的SteamID分组**
- 点击页面底部的 **"新建SteamID分组"** 卡片
- 输入新的SteamID
- 选择数据源类型并配置

## Steam市场数据源

Steam市场提供两种模式：

### 模式1：Steam市场（未登录）

**适用场景：** 快速获取公开市场价格，无需登录

**配置步骤：**
1. 选择数据源类型：**Steam市场**
2. 填写 **数据源名称**（例如：Steam市场-公开）
3. 点击 **"保存"**

> 💡 **提示：** 未登录模式可能受到Steam访问限制，建议使用登录模式。

### 模式2：Steam市场（登录）

**适用场景：** 需要更稳定的数据采集，避免频率限制

**配置步骤：**
1. 选择数据源类型：**Steam市场(登录)**
2. 填写 **数据源名称**（例如：Steam市场-已登录）
3. 填写 **SteamID**
4. 填写 **Cookie**（从浏览器获取）
5. 点击 **"保存"**

**如何获取Cookie：**
1. 在浏览器中登录 [steamcommunity.com](https://steamcommunity.com)
2. 按 `F12` 打开开发者工具
3. 切换到 **"网络"** 或 **"Network"** 标签
4. 刷新页面，找到任意请求
5. 在请求头中找到 **Cookie** 字段并复制

## 网易BUFF数据源

**适用场景：** 获取国内BUFF平台的饰品价格

> ⚠️ **重要：** 此功能需要使用MuMu模拟器和CA证书来抓取APP数据，请确保已完成[先决条件](#先决条件)中的配置。

### 配置步骤

**步骤1：点击"获取BUFF令牌"按钮**

系统会启动代理服务器，等待手机APP访问。

**步骤2：配置MuMu模拟器网络代理**

1. 在MuMu模拟器中打开 **设置 → 网络和互联网 → WLAN**
2. 长按已连接的WiFi，选择 **"修改网络"**
3. 展开 **"高级选项"**
4. 代理设置选择 **"手动"**
5. 填入页面显示的代理地址（例如：`192.168.1.100:8888`）
6. 保存设置

**步骤3：在MuMu模拟器中打开BUFF APP并登录**

1. 在MuMu模拟器中打开 **网易BUFF APP**
2. 登录您的账号
3. 系统会自动捕获令牌信息
4. 看到 **"✓ 令牌获取成功!"** 提示后，关闭模拟器代理

**步骤4：填写基础配置**

展开 **"基础配置"** 折叠面板：
- **SteamID**：您的Steam ID
- **Cookie**：BUFF网站的Cookie（可选，用于网页端）

**步骤5：填写应用信息配置**

展开 **"应用信息配置"** 折叠面板：
- **app-version**：APP版本号（例如：3.5.0）
- **app-version-code**：版本代码（例如：350）
- **channel**：渠道标识（例如：google）
- **device-id**：设备ID
- **device-model**：设备型号（例如：Pixel 5）

> 💡 **提示：** 这些信息在获取令牌时会自动填充，通常无需手动修改。

**步骤6：填写设备信息配置**

展开 **"设备信息配置"** 折叠面板：
- **x-client-ip**：客户端IP
- **x-real-ip**：真实IP

**步骤7：保存配置**

点击 **"保存"** 按钮完成配置。

> ⚠️ **重要：** 此功能需要使用MuMu模拟器和CA证书来抓取APP数据，请确保已完成[先决条件](#先决条件)中的配置。

系统会启动代理服务器，等待手机APP访问。

**步骤2：配置MuMu模拟器网络代理**

1. 在MuMu模拟器中打开 **设置 → 网络和互联网 → WLAN**
2. 长按已连接的WiFi，选择 **"修改网络"**
3. 展开 **"高级选项"**
4. 代理设置选择 **"手动"**
5. 填入页面显示的代理地址（例如：`192.168.1.100:8888`）
6. 保存设置

**步骤3：在MuMu模拟器中打开悠悠有品APP并登录**

1. 在MuMu模拟器中打开 **悠悠有品APP**
2. 登录您的账号
3. 系统会自动捕获令牌信息
4. 看到 **"✓ 令牌获取成功!"** 提示后，关闭模拟器代理

**步骤4：填写基础配置**

- **手机号**：悠悠有品账号绑定的手机号

> 💡 **提示：** 这些密钥信息在获取令牌时会自动填充，请勿随意修改。

点击 **"保存"** 按钮完成配置。

## 管理数据源

### 查看数据源

所有数据源按 **SteamID** 分组显示，每个分组显示：
- SteamID信息
- 该分组下的数据源数量
- 数据源卡片列表

每个数据源卡片显示：
- 数据源名称
- 数据源类型标签
- 操作按钮（编辑、采集）

### 编辑数据源

1. 找到要编辑的数据源卡片
2. 点击 **"编辑"** 按钮
3. 在弹出的对话框中修改配置
4. 点击 **"保存"** 保存更改

> ⚠️ **注意：** 数据源类型创建后无法修改，如需更换类型请删除后重新创建。

### 删除数据源

1. 点击数据源的 **"编辑"** 按钮
2. 在编辑对话框底部找到 **"删除数据源"** 按钮
3. 点击后会弹出确认对话框
4. 确认删除

> ⚠️ **警告：** 删除操作不可恢复，请谨慎操作！

## 采集数据

### 手动采集

1. 找到要采集的数据源卡片
2. 点击 **"采集"** 按钮
3. 系统会开始采集该数据源的价格信息
4. 采集过程中按钮显示 **"采集中..."**
5. 采集完成后会显示提示信息

> 💡 **提示：** 完美世界APP数据源不支持手动采集，系统会自动同步。

### 采集状态

- **采集中...** - 正在从平台获取数据
- **采集完成** - 数据已成功获取并保存
- **采集失败** - 请检查配置是否正确

### 采集建议

1. **首次配置后立即采集** - 验证配置是否正确
2. **定期采集** - 保持价格数据的时效性
3. **错峰采集** - 避免在平台高峰期采集
4. **多源对比** - 配置多个数据源，获取更全面的价格信息

## 常见问题

### Q1: 为什么需要配置多个数据源？

**A:** 不同平台的价格可能存在差异，配置多个数据源可以：
- 获取更全面的市场价格信息
- 对比不同平台的价格差异
- 提高数据采集的稳定性（某个平台故障时可用其他源）

### Q2: 获取令牌时一直显示"等待手机APP访问"？

**A:** 请检查：
1. MuMu模拟器是否正在运行
2. 模拟器网络代理配置是否正确（IP地址和端口）
3. CA证书是否已正确安装（参考[先决条件](#先决条件)）
4. 防火墙是否阻止了代理端口
5. APP是否已正确登录

### Q3: 令牌获取成功后还能正常使用吗？

**A:** 令牌有有效期，通常为：
- **BUFF令牌**：约30天
- **悠悠有品令牌**：约30天

令牌过期后需要重新获取。建议定期检查采集是否正常，如失败请重新获取令牌。

### Q4: Steam Cookie如何获取？

**A:** 
1. 浏览器登录Steam社区
2. 按F12打开开发者工具
3. 切换到Network标签
4. 刷新页面
5. 点击任意请求，在Headers中找到Cookie字段
6. 复制完整的Cookie值

### Q5: 采集失败怎么办？

**A:** 请按以下步骤排查：
1. 检查网络连接是否正常
2. 检查配置信息是否正确（特别是令牌和Cookie）
3. 如果是BUFF或悠悠有品，检查CA证书是否正常工作
4. 尝试重新获取令牌
5. 查看系统日志获取详细错误信息
6. 如果是Steam数据源，可能是访问频率限制，请稍后再试

### Q6: 可以同时采集多个数据源吗？

**A:** 可以，但建议：
- 避免同时采集过多数据源（建议不超过3个）
- 给每次采集之间留出适当间隔（建议5-10秒）
- 避免频繁采集导致被平台限制

### Q7: 为什么必须使用MuMu模拟器？

**A:** 
- MuMu模拟器提供了稳定的Android环境
- 支持ROOT权限，便于安装CA证书
- 可以方便地配置网络代理
- 便于自动化操作和数据采集

如果您使用真实手机，也可以实现相同功能，但需要：
- 手机已ROOT
- 已安装CA证书到系统目录
- 手机和电脑在同一局域网

### Q8: CA证书安装失败怎么办？

**A:** 请参考 [`Documents\其他功能\开荒工具\ADB与网络证书安装.md`](../其他功能/开荒工具/ADB与网络证书安装.md) 文档中的常见问题部分，或重新按照文档步骤操作。
