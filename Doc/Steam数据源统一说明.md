# Steam数据源统一功能说明

## 📋 概述

Steam数据源现在统一为一个类型，支持**三种Cookie获取方式**，提供更灵活的配置选项。

## ✨ 功能特性

### 1️⃣ 三种Cookie获取方式

#### 方式一：扫码登录（推荐）
- ✅ 最安全便捷
- ✅ 无需输入密码
- ✅ 支持Steam Guard
- 📱 使用Steam手机APP扫码即可

**使用流程：**
```
1. 选择"扫码登录"
2. 点击二维码区域获取二维码
3. 使用Steam手机APP扫描
4. 确认登录
5. 自动获取Cookie
6. 手动输入SteamID
7. 提交
```

#### 方式二：账号密码登录
- ✅ 传统登录方式
- ✅ 支持Steam Guard验证码
- 🔐 密码加密传输

**使用流程：**
```
1. 选择"账号密码登录"
2. 输入Steam用户名
3. 输入Steam密码
4. 输入Steam Guard验证码（如需要）
5. 点击"立即登录获取Cookie"
6. 等待登录成功
7. 手动输入SteamID
8. 提交
```

#### 方式三：手动输入
- ✅ 完全手动控制
- ✅ 适合已有Cookie的用户
- 📋 直接粘贴Cookie字符串

**使用流程：**
```
1. 选择"手动输入"
2. 在Cookie文本框中粘贴Cookie字符串
3. 输入SteamID
4. 提交
```

## 📝 编辑数据源

编辑对话框与新增对话框**完全相同**，支持：
- ✅ 查看当前Cookie获取方式
- ✅ 切换到其他获取方式
- ✅ 重新获取Cookie（当Cookie失效时）
- ✅ 更新SteamID和其他配置

### Cookie失效处理流程

```
情况：Cookie已过期，需要重新获取

步骤：
1. 打开编辑对话框
2. 切换Cookie获取方式（可选）
   - 从"手动输入" → "扫码登录"（更方便）
   - 从"账号密码" → "扫码登录"（更安全）
3. 重新获取Cookie
4. 确认SteamID无误
5. 保存
```

## 🔄 兼容性说明

### Steam市场(登录) 类型

旧的`steam_login`类型数据源**完全兼容**：
- ✅ 编辑时自动识别Cookie获取方式
- ✅ 有用户名密码 → 自动选择"账号密码登录"
- ✅ 无用户名密码 → 自动选择"手动输入"
- ✅ 可在编辑时切换到其他方式
- ✅ 保存后数据格式统一

### 数据迁移

```
旧数据 (steam_login)          新数据 (steam)
├─ cookies               →    ├─ cookies
├─ steamID               →    ├─ steamID
├─ steamUsername         →    ├─ steamCookieMethod (新增)
├─ steamPassword         →    ├─ steamUsername (可选)
└─ updateFreq            →    ├─ steamPassword (可选)
                               └─ updateFreq

自动识别逻辑：
- 如果有 steamCookieMethod → 使用该值
- 如果有 steamUsername → 判定为 "password"
- 其他情况 → 判定为 "manual"
```

## 💾 配置数据结构

### 新增数据源配置

```json
{
  "dataName": "我的Steam市场",
  "type": "steam",
  "enabled": true,
  "configJson": {
    "cookies": "steamLoginSecure=xxxxx; sessionid=xxxxx",
    "steamID": "76561198123456789",
    "steamCookieMethod": "qrcode",  // 或 "password" 或 "manual"
    "steamUsername": "",             // password方式时有值
    "steamPassword": "",             // password方式时有值
    "updateFreq": "15min",
    "sleep_time": "6000"
  }
}
```


## 🎨 UI交互细节

### 扫码登录交互

1. **初始状态**
   - 显示网格图标
   - 提示"点击获取Steam登录二维码"
   - 鼠标悬停时背景变深

2. **加载状态**
   - 显示旋转的加载图标
   - 提示"正在获取二维码..."

3. **显示二维码**
   - 显示二维码图片（200x200）
   - 提示"请使用Steam手机APP扫描二维码"
   - 显示状态标签：等待扫码/扫码成功/已过期

4. **过期处理**
   - 显示"已过期"标签（橙色）
   - 显示"点击刷新二维码"链接
   - 点击后重新生成二维码

5. **成功状态**
   - 显示"扫码成功"标签（绿色）
   - 二维码区域不可再点击
   - Cookie自动填充

### 账号密码登录交互

1. **输入表单**
   - 用户名：普通输入框
   - 密码：密码输入框，支持显示/隐藏
   - Steam Guard：可选，5位数字限制

2. **登录按钮**
   - 未登录：显示"立即登录获取Cookie"
   - 登录中：显示"登录中..."并显示加载动画
   - 登录成功：按钮恢复，Cookie自动填充

3. **状态提示**
   - 成功：绿色提示框
   - 失败：橙色警告框
   - 包含详细信息

### 手动输入交互

1. **文本域**
   - 3行高度
   - 支持多行粘贴
   - 占位符提示格式

2. **验证**
   - 提交时检查是否为空
   - 提示用户补充

## 🚀 使用建议

### 首次配置推荐

1. **优先使用扫码登录**
   - 最安全便捷
   - 无需担心密码泄露
   - 支持Steam Guard

2. **备选账号密码**
   - 扫码不方便时使用
   - 需要批量配置时使用

3. **手动输入**
   - 已有现成Cookie时使用
   - 从其他工具导入时使用

### Cookie失效时

```
推荐处理流程：
1. 编辑数据源
2. 切换到"扫码登录"（最快捷）
3. 扫码获取新Cookie
4. 保存

替代方案：
1. 编辑数据源
2. 保持"账号密码登录"
3. 重新输入密码和验证码
4. 点击"重新登录获取Cookie"
5. 保存
```

### 安全建议

- ⚠️ **不建议**在生产环境明文保存账号密码
- ✅ **推荐**使用扫码登录或手动输入方式
- 🔐 定期更换Cookie（设置合适的更新频率）
- 📝 记录SteamID，避免每次手动输入

## ❓ 常见问题

### Q1: steam_login类型还能用吗？
**A**: 可以！旧的steam_login类型数据源完全兼容，编辑时会自动显示三种Cookie获取方式，可以随时切换。

### Q2: 如何从手动输入切换到扫码登录？
**A**: 编辑数据源 → 选择"扫码登录" → 点击二维码区域 → 扫码 → 保存。

### Q3: 扫码登录后需要手动输入SteamID吗？
**A**: 是的，为了确保准确性，SteamID需要手动输入。

### Q4: Cookie失效了怎么办？
**A**: 编辑数据源，选择任意一种方式重新获取Cookie即可，推荐使用扫码登录（最快捷）。

### Q5: 三种方式有什么区别？
**A**: 
- 扫码登录：最安全便捷，无需输入密码
- 账号密码：传统方式，需要输入密码和验证码
- 手动输入：适合已有Cookie的用户

### Q6: 为什么账号密码方式不推荐？
**A**: 明文保存密码存在安全风险，推荐使用扫码登录或仅在必要时手动更新Cookie。

### Q7: 可以同时配置多个Steam数据源吗？
**A**: 不可以，每种类型只能配置一个数据源。
