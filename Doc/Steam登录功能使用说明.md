# Steam登录功能使用说明

## 📝 概述

Steam登录功能已集成到数据源管理页面，支持通过Steam用户名和密码自动登录获取Cookie，无需手动从浏览器复制Cookie。

## ✨ 功能特性

1. **自动登录** - 输入Steam用户名和密码即可自动获取Cookie
2. **Steam Guard支持** - 支持Steam Guard手机令牌验证
3. **自动刷新** - Cookie失效时可一键重新登录
4. **安全存储** - 登录凭证安全存储在本地数据库

## 🚀 使用步骤

### 添加新的Steam数据源（使用登录）

1. **打开数据源管理页面**
   - 进入系统后点击"数据源管理"

2. **创建新数据源**
   - 点击"新建SteamID分组"或在现有分组中点击"添加数据源"

3. **选择数据源类型**
   - 在"数据源类型"下拉菜单中选择 **"Steam市场(登录)"**

4. **填写登录信息**
   - **数据源名称**: 自定义名称，如"我的Steam账号"
   - **Steam用户名**: 您的Steam登录用户名
   - **Steam密码**: 您的Steam登录密码
   - **Steam Guard验证码**: 如果启用了Steam Guard手机令牌，请输入5位验证码
   - **SteamID**: 您的Steam ID（如：76561198XXXXXXXXX）
   - **更新频率**: 选择数据更新频率

5. **点击登录按钮**
   - 点击 **"立即登录获取Cookie"** 按钮
   - 系统会自动使用您的凭证登录Steam
   - 登录成功后会显示绿色提示

6. **保存数据源**
   - 登录成功后点击"添加数据源"保存配置

### 编辑现有Steam登录数据源

1. **找到数据源**
   - 在数据源列表中找到对应的Steam登录数据源
   - 点击"编辑"按钮

2. **更新登录信息**
   - 如需更新密码或其他信息，请修改对应字段
   - 如果Cookie失效，点击 **"重新登录获取Cookie"** 按钮

3. **保存更改**
   - 点击"保存更改"按钮

## ⚠️ 常见问题

### Q1: 登录时提示"需要Steam Guard验证码"怎么办？

**A:** 
1. 打开您的Steam Guard移动应用
2. 查看当前的5位验证码
3. 在"Steam Guard验证码"输入框中输入验证码
4. 重新点击"立即登录获取Cookie"按钮

### Q2: 登录时提示"需要邮箱验证码"怎么办？

**A:** 
1. Steam已向您的注册邮箱发送了验证码
2. 查收邮件并获取验证码
3. 暂时不支持邮箱验证码输入，建议：
   - 改用手动Cookie方式
   - 或在Steam中启用Steam Guard移动验证器

### Q3: 登录时提示"需要图形验证码"怎么办？

**A:** 
当前版本暂不支持图形验证码识别，建议：
- 等待几分钟后重试
- 或改用手动Cookie方式（选择"Steam市场"类型）

### Q4: 如何获取我的Steam ID？

**A:** 
1. 访问 https://steamcommunity.com
2. 登录后点击您的用户名
3. 查看个人资料页面的URL
4. URL中的数字就是您的Steam ID
   - 例如：`https://steamcommunity.com/profiles/76561198XXXXXXXXX/`

### Q5: 密码会被明文存储吗？

**A:** 
- 密码存储在本地数据库中
- 建议使用强密码
- 定期更换密码以确保安全

### Q6: Cookie会过期吗？

**A:** 
- Steam Cookie会定期过期
- 当Cookie失效时，系统会提示
- 使用"重新登录获取Cookie"功能可快速更新

### Q7: 与手动Cookie方式有什么区别？

**A:** 

| 特性 | Steam市场(登录) | Steam市场(手动Cookie) |
|------|----------------|---------------------|
| 获取Cookie | 自动 | 手动从浏览器复制 |
| 更新难度 | 简单（一键刷新） | 需重新复制 |
| Steam Guard | 支持手机令牌 | 需在浏览器中完成 |
| 安全性 | 密码存储 | 仅存储Cookie |

## 🔐 安全建议

1. **使用强密码** - 确保Steam账号密码足够复杂
2. **启用Steam Guard** - 建议使用Steam Guard移动验证器
3. **定期更换** - 定期更改Steam密码
4. **专用账号** - 建议使用专门的Steam账号用于数据采集
5. **本地部署** - 确保系统部署在安全的本地环境中

## 📊 工作流程

```
1. 用户输入Steam凭证
   ↓
2. 系统发送登录请求到Spider服务
   ↓
3. Spider调用Steam登录API
   ↓
4. 处理RSA加密、Steam Guard等
   ↓
5. 获取Session Cookie
   ↓
6. 返回Cookie给前端
   ↓
7. 保存配置到数据库
```

## 🛠️ 技术架构

- **前端**: Vue 3 + Element Plus
- **后端Spider**: Flask Blueprint
- **登录模块**: Python + RSA加密
- **存储**: SQLite数据库

## 📝 API端点

### 登录接口
- **URL**: `http://localhost:9002/steamLoginV1/login`
- **方法**: POST
- **参数**:
  ```json
  {
    "username": "steam用户名",
    "password": "密码",
    "twofactor_code": "5位验证码(可选)"
  }
  ```

### 验证接口
- **URL**: `http://localhost:9002/steamLoginV1/verify`
- **方法**: POST
- **参数**:
  ```json
  {
    "steam_id": "Steam ID",
    "cookies": "Cookie字符串"
  }
  ```
