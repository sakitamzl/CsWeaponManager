## 🧪 测试步骤

### 方法一：扫码登录（推荐）

1. **打开数据源管理页面**
   - 访问：http://localhost:9003/
   - 进入"数据源管理"

2. **新增数据源**
   - 点击"新增数据源"按钮
   - 数据源名称：任意填写（如"我的Steam库存"）
   - 数据源类型：选择"Steam市场(登录)"

3. **选择扫码登录**
   - 登录方式：选择"扫码登录（推荐）"

4. **生成二维码**
   - 点击"生成二维码"按钮
   - 等待二维码图片显示（应该会显示一个黑白二维码）

5. **使用Steam手机APP扫码**
   - 打开Steam手机APP
   - 找到扫码功能（通常在设置或菜单中）
   - 扫描页面上的二维码
   - 在手机上点击"确认"或"登录"

6. **等待登录成功**
   - 页面会每2秒自动检查一次登录状态
   - 状态显示：
     - 🟡 "等待扫码中..." - 等待用户扫码
     - 🟢 "✅ 登录成功" - 扫码成功
     - 🔴 "❌ 二维码已过期" - 超过5分钟未扫码
   - 登录成功后，Cookie和SteamID会自动填充

7. **完成配置**
   - 选择更新频率
   - 点击"提交"保存数据源

### 方法二：账号密码登录

1. **选择密码登录**
   - 登录方式：选择"账号密码登录"

2. **填写账号信息**
   - Steam用户名：输入你的Steam账号
   - Steam密码：输入密码
   - Steam Guard验证码：如果启用了手机令牌，输入5位验证码

3. **点击登录**
   - 点击"立即登录获取Cookie"按钮
   - 等待登录完成

## ❗ 常见问题

### Q1: 点击"生成二维码"后没有反应？
**A:** 检查：
1. 查看浏览器控制台（F12）是否有报错
2. 查看Spider后端日志
3. 确认Spider服务正在运行
4. 确认网络可以访问 api.steampowered.com

### Q2: 提示"qrcode库未安装"？
**A:** 运行：
```bash
pip install qrcode[pil]
```

### Q3: 二维码生成成功但一直显示"等待扫码中"？
**A:** 检查：
1. 是否真的用Steam APP扫了码
2. 扫码后是否在手机上点击了"确认"
3. 等待时间是否超过5分钟（二维码会过期）
4. 查看后端日志中的轮询响应

### Q4: 提示"获取cookies失败"？
**A:** 这是正常情况，Steam的新版API可能返回的是token而不是传统cookies。检查日志看是否有返回access_token。

### Q5: 账号密码登录提示"获取RSA密钥失败"？
**A:** 检查网络连接，确保可以访问 steamcommunity.com
