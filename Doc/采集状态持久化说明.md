# 采集状态持久化功能说明

## 功能概述

采集状态持久化功能确保了在用户切换页面或刷新浏览器后，如果数据采集任务仍在进行中（后端API未返回），采集按钮会继续保持"采集中"状态，防止重复提交采集任务。

## 实现方式

### 1. 持久化存储

使用 `localStorage` 来存储正在采集的数据源ID列表：
- **存储键**: `cs_weapon_manager_collecting_sources`
- **存储格式**: JSON数组，包含所有正在采集的数据源ID

### 2. 状态管理 Composable

创建了 `useCollectionState.js` composable 来统一管理采集状态：

```javascript
// WebSite/src/composables/useCollectionState.js
export function useCollectionState() {
  return {
    collectingSourceIds,    // 响应式的采集ID集合
    startCollecting,        // 开始采集
    stopCollecting,         // 停止采集
    isCollecting,          // 检查是否正在采集
    clearAllCollecting     // 清空所有采集状态
  }
}
```

### 3. 全局单例模式

使用全局单例确保所有组件共享同一个采集状态，即使在不同页面之间切换也能保持状态一致。

## 使用方法

### 在组件中使用

```javascript
import { useCollectionState } from '@/composables/useCollectionState.js'

setup() {
  const { collectingSourceIds, startCollecting, stopCollecting } = useCollectionState()
  
  const startCollection = async (source) => {
    // 开始采集
    startCollecting(source.dataID)
    
    try {
      // 调用后端API
      await axios.post(...)
    } finally {
      // 无论成功还是失败，都要停止采集状态
      stopCollecting(source.dataID)
    }
  }
}
```

### 按钮状态绑定

```vue
<el-button 
  @click="startCollection(source)" 
  :disabled="!source.enabled || collectingSourceIds.has(source.dataID)"
  :loading="collectingSourceIds.has(source.dataID)"
>
  {{ collectingSourceIds.has(source.dataID) ? '采集中' : '采集' }}
</el-button>
```

## 功能特性

### 1. ✅ 自动持久化

每次状态变化都会自动保存到 localStorage，无需手动调用保存方法。

### 2. ✅ 页面刷新保持状态

刷新浏览器后，正在采集的任务状态会自动恢复：
- 按钮保持"采集中"状态
- 按钮被禁用，防止重复点击
- 等待后端API返回后自动恢复

### 3. ✅ 多页面共享状态

在不同页面之间切换时，采集状态保持一致：
- 从数据源页面切换到其他页面
- 再切换回数据源页面
- 采集状态依然保持

### 4. ✅ 错误处理

使用 `try-catch-finally` 结构确保：
- 无论API调用成功还是失败
- 无论网络连接是否正常
- 都会在 finally 中调用 `stopCollecting()` 恢复按钮状态

## 应用场景

### 场景1：列表中的采集按钮
用户点击某个数据源的"采集"按钮后：
1. 按钮立即显示为"采集中"
2. 按钮被禁用
3. 用户切换到其他页面
4. 再返回数据源页面
5. **按钮仍然显示"采集中"状态**
6. 等后端返回后自动恢复

### 场景2：全部采集按钮
用户在编辑对话框点击"全部采集"按钮后：
1. "全部采集"按钮显示为"采集中"
2. 列表中对应数据源的"采集"按钮也显示为"采集中"
3. 两个按钮都被禁用
4. 用户切换页面或刷新浏览器
5. **两个按钮都保持"采集中"状态**
6. 等后端返回后同时恢复

### 场景3：浏览器意外关闭
如果用户在采集过程中意外关闭浏览器：
1. 再次打开网站时，采集状态会从 localStorage 恢复
2. 如果后端已经完成采集但客户端未收到响应
3. 按钮会一直保持"采集中"状态
4. 用户需要手动刷新页面或等待下次后端响应

## 注意事项

### 1. 清理僵尸状态

如果出现以下情况，可能需要手动清理采集状态：
- 后端服务重启，导致采集任务中断
- 网络长时间中断，API无法返回
- 浏览器缓存损坏

**清理方法**：
```javascript
// 在浏览器控制台执行
localStorage.removeItem('cs_weapon_manager_collecting_sources')
// 或者
clearAllCollecting()
```

### 2. 后端API超时处理

建议后端API设置合理的超时时间（如30秒），确保：
- 无论采集成功还是失败都要返回响应
- 避免客户端无限等待

### 3. 数据源ID的唯一性

确保每个数据源的 `dataID` 是全局唯一的，避免状态混淆。

## 技术细节

### localStorage 存储示例

```json
["dataID_1", "dataID_2", "dataID_3"]
```

### 状态同步机制

1. **初始化**: 页面加载时从 localStorage 读取状态
2. **更新**: 调用 `startCollecting()` 或 `stopCollecting()` 时自动保存
3. **监听**: 使用 Vue 的 `watch` 监听状态变化（对于复杂场景的备份）
4. **手动触发**: Set 的 add/delete 方法不会触发 deep watch，因此手动调用保存
